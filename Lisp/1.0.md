>Подстановочная модель применения процедуры 

Заменяем каждый формальный параметр соответствующим аргументом. Интерпретатор сначала вычисляет оператор и операнды, а затем применяет процедуру к ним.
	*(f 5) -> (sum-of-squares (+ 5 1) (* *5 2)) -> (+ (* *6 6) (*10 10))  
-это не является представлением реальной работы интерпретатора-

>Аппликативный и нормальный порядки вычисления

- Аппликативная модель (applicative-order evaluation)
	Эта модель не вычисляет аргументы, пока их значение не понадобится 
- Нормальная модель (normal-oreder evaluation)
	Эта модель сначала вычисляет аргументы, а потом применяет процедуру к ним
Программа для проверки, какую модель использует интерпретатор:
```
(define (p) (p))

(define (test x y)   
	(if ( = x 0) 0 y)
	)

test 0 p
```

>Условные выражения и предикаты

- Разбор случаев (case analysis)
```
(cond (<p1> <e1>)
	  (<p2> <e2>)
	  ...
	  (<pn> <en>))
```
- Условное выражение
	*( if <предикат> <следствие> <альтернатива> )*
```
(define (abs x)
	(if (<x 0)
		(- x)
		x))
```
- Операции логической композиции
	- ```(and <e1> ... <en>)```
	- ```(or <e1> ... <en>)```
	- ```(not <e1>)```
